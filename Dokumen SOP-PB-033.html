## TUJUAN

Prosedur ini bertujuan untuk menstandarisasikan proses backup dan keamanan data pembelian guna memastikan:

Ketersediaan data pembelian yang kontinu dan dapat diakses kapan saja
Perlindungan data sensitif pembelian dari ancaman keamanan siber
Pemulihan data yang cepat dan efektif dalam situasi darurat
Kepatuhan terhadap regulasi perlindungan data dan privasi
Integritas dan keaslian data pembelian dari modifikasi tidak sah

Penting: Implementasi sistem backup dan keamanan yang efektif dapat mencegah kerugian hingga 95% dari potensi kehilangan data dan mengurangi downtime sistem hingga 80% melalui strategi yang komprehensif.

## DEFINISI

RTO (Recovery Time Objective)

Waktu maksimal yang diterima untuk pemulihan sistem

RPO (Recovery Point Objective)

Point waktu maksimal data yang boleh hilang

Encryption at Rest

Enkripsi data yang disimpan dalam media penyimpanan

Zero Trust Architecture

Model keamanan yang tidak mempercayai siapa pun

Istilah Teknis Keamanan Data:

  • DRP: Disaster Recovery Plan
  • BCP: Business Continuity Plan
  • SLA: Service Level Agreement
  • RAID: Redundant Array of Independent Disks
  • MFA: Multi-Factor Authentication
## ALUR PROSES BACKUP DAN KEAMANAN DATA PEMBELIAN
1
Identifikasi dan Klasifikasi Data

Mengidentifikasi dan mengklasifikasikan data pembelian berdasarkan sensitivitas dan criticality

Klasifikasi data: Public, Internal, Confidential, Restricted, Highly Sensitive

2
Penetapan RTO dan RPO

Menetapkan Recovery Time Objective dan Recovery Point Objective untuk setiap kategori data

  • Critical data: RTO < 4 jam, RPO < 15 menit
  • Important data: RTO < 8 jam, RPO < 1 jam
  • Standard data: RTO < 24 jam, RPO < 4 jam
  • Archive data: RTO < 72 jam, RPO < 24 jam
3
Implementasi Backup Strategy

Menerapkan strategi backup sesuai dengan klasifikasi data dan RTO/RPO

Backup methods: Full backup, incremental backup, differential backup, snapshot backup

4
Enkripsi dan Pengamanan Data

Mengimplementasikan enkripsi data dan mekanisme keamanan lainnya

Security measures: AES-256 encryption, SSL/TLS, VPN, firewall, intrusion detection

5
Monitoring dan Alert System

Memantau sistem backup dan keamanan dengan real-time alert

  • Backup success/failure monitoring
  • Security breach detection
  • System performance tracking
  • Anomaly behavior detection
  • Automated alert escalation
6
Regular Backup Validation

Melakukan validasi rutin terhadap backup data untuk memastikan integritas

Validation methods: Checksum verification, test restoration, data integrity check

7
Disaster Recovery Testing

Melakukan testing pemulihan bencana secara berkala

Testing scenarios: Full system recovery, partial recovery, data corruption recovery, cyber attack recovery

8
Access Control dan Authentication

Mengelola akses data dan sistem autentikasi pengguna

Access management: Role-based access control, multi-factor authentication, single sign-on, privileged access management

9
Audit dan Compliance Monitoring

Melakukan audit rutin dan monitoring kepatuhan regulasi

Compliance requirements: GDPR, PDP Law, ISO 27001, SOX, Industry regulations

10
Continuous Improvement

Melakukan evaluasi dan peningkatan berkelanjutan sistem backup dan keamanan

Improvement activities: Technology update, process optimization, threat intelligence, security training

## STRATEGI BACKUP DAN KEAMANAN DATA

Klasifikasi strategi backup berdasarkan jenis data dan criticality:

Kategori Data Frekuensi Backup Retensi Period Storage Location Encryption Level
Critical Transaction Data Real-time 7 tahun On-premise + Cloud AES-256
Supplier Master Data Hourly 10 tahun On-premise + Cloud AES-256
Contract Documents Daily 15 tahun Cloud + Offline AES-256 + PGP
Analytical Data Weekly 5 tahun Cloud AES-128
Archive Data Monthly Permanent Offline Storage AES-256

Prinsip 3-2-1 Backup Strategy:

  • 3 Copies: Minimal 3 copy data (production + 2 backup)
  • 2 Different Media: Backup disimpan di 2 media berbeda (disk + tape/cloud)
  • 1 Off-site: Minimal 1 copy disimpan di lokasi terpisah

Lapisan Keamanan Data Pembelian:

Physical Security: Data center access control, CCTV, biometric authentication, environmental controls

Network Security: Firewall, IDS/IPS, VPN, network segmentation, DDoS protection

Application Security: Secure coding, vulnerability assessment, penetration testing, API security

Data Security: Encryption, data masking, tokenization, data loss prevention

Access Security: RBAC, MFA, privileged access management, session management
