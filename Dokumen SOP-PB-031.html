## TUJUAN

Prosedur ini bertujuan untuk menstandarisasikan penggunaan sistem purchasing software guna memastikan:

Efisiensi proses procurement melalui otomatisasi dan digitalisasi
Konsistensi data dan informasi dalam seluruh proses purchasing
Transparansi dan akuntabilitas dalam pengelolaan pengadaan
Kepatuhan terhadap kebijakan dan prosedur perusahaan
Optimalisasi penggunaan fitur dan modul sistem untuk meningkatkan produktivitas

Penting: Implementasi sistem purchasing software yang efektif dapat mengurangi waktu proses procurement hingga 60% dan meningkatkan akurasi data hingga 95% melalui otomatisasi dan standarisasi.

## RUANG LINGKUP

Prosedur ini mencakup seluruh aspek penggunaan sistem purchasing software:

User Management

Pengelolaan akses user dan role-based permissions

Purchase Requisition

Proses pengajuan dan approval permintaan pembelian

Purchase Order

Pembuatan dan pengelolaan purchase order digital

Reporting & Analytics

Pelaporan dan analisis data purchasing

Penanggung Jawab:

System Administrator: Pengelolaan sistem dan user management
Purchasing Manager: Supervisi penggunaan sistem dan approval workflow
Purchasing Officer: Operasional harian sistem purchasing
IT Support: Technical support dan maintenance sistem
Department User: Pengguna sistem untuk pengajuan kebutuhan
Finance Officer: Integrasi sistem dengan modul finance

## ALUR PROSES PENGGUNAAN SISTEM PURCHASING SOFTWARE
1
User Registration dan Authentication

Proses registrasi user dan verifikasi identitas untuk akses sistem

Proses authentication: Single Sign-On (SSO), Two-Factor Authentication (2FA), role-based access control

2
Pengisian Purchase Requisition (PR)

User department mengisi form PR digital dengan data yang lengkap

  • Item description dan specification
  • Quantity dan unit of measure
  • Required date dan delivery location
  • Budget code dan cost center
  • Justification dan supporting document
3
Digital Approval Workflow

Proses approval PR melalui workflow otomatis sesuai hierarki

Approval routing: Department Head → Budget Controller → Purchasing Manager → Director (jika diperlukan)

4
Supplier Selection dan Quotation Management

Proses pemilihan supplier dan pengelolaan penawaran harga

Fitur supplier management: Supplier database, performance rating, quotation comparison, historical data

5
Purchase Order Creation

Pembuatan PO digital dengan template standar perusahaan

  • Auto-generated PO number
  • Standard terms and conditions
  • Digital signature
  • Auto-calculated amount
  • Attachment management
6
PO Approval dan Distribution

Approval PO final dan distribusi ke supplier melalui sistem

Distribution channels: Email automation, supplier portal download, printed copy (jika diperlukan)

7
Order Tracking dan Monitoring

Pemantauan status order dan pengiriman secara real-time

Tracking features: Order status, delivery timeline, milestone tracking, exception alert

8
Goods Receipt dan Quality Check

Pencatatan penerimaan barang dan hasil quality inspection

GRN process: Barcode scanning, quantity verification, quality inspection recording, acceptance/rejection

9
Invoice Processing dan Payment

Proses matching invoice dengan PO dan GRN untuk pembayaran

3-way matching: PO vs Invoice vs GRN, exception handling, payment approval workflow

10
Reporting dan Performance Analysis

Generasi laporan dan analisis kinerja procurement

Reporting capabilities: Custom reports, dashboard analytics, performance metrics, trend analysis

## MODUL SISTEM DAN FITUR UTAMA

Klasifikasi modul sistem purchasing software dan fungsionalitas utama:

Modul Sistem Fungsi Utama User Access Integration
Requisition Management Pengajuan dan approval PR All Users Budget System
Supplier Management Database supplier dan performance Purchasing Team CRM System
Purchase Order Pembuatan dan tracking PO Purchasing Team Inventory System
Contract Management Pengelolaan kontrak supplier Manager Level Legal System
Analytics & Reporting Laporan dan dashboard All Managers BI Tools

Critical System Features:

  • Mobile Accessibility: Akses sistem melalui mobile device
  • Real-time Notification: Notifikasi otomatis untuk setiap status change
  • Document Management: Penyimpanan dan retrieval dokumen digital
  • Audit Trail: Pencatatan lengkap semua aktivitas sistem
  • Custom Workflow: Kemampuan kustomisasi alur approval

Security dan Access Control:

User Roles: Super Admin, Purchasing Manager, Purchasing Officer, Department User, Finance User, Read-only User

Access Levels: Full Access, Create & Edit, View Only, No Access

Data Security: Encryption, Regular Backup, Access Log, Session Timeout

Compliance: GDPR Compliance, Data Privacy, Audit Requirement
